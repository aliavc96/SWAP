En el caso de Apache, se utiliza como servidor web HTTP para la creación de páginas y servicios web. Además es gratuito. 
Esté servidor web se encuentra alojado en un ordenador que cuenta con conexión a Internet. El web server se encuentra a la espera 
de que algún navegador le haga alguna petición, como por ejemplo acceder a una página web y resonde a la petición enviando el 
código HTML mediante la transferencia de datos en red.

Nginx es un servidor web/proxy inverso ligero y de alto rendimiento y un proxy (que hace de intermediario en las peticiones de 
recursos que realiza un cliente (A) a otro servidor (C)) para protocolos de correo electrónico.
Es de código abierto y multiplataforma, por lo que puede ser utilizado en sistemas tanyto Windows como Unix. Es utilizado por 
Netflix, GitHub, SourceForge...

Thttpd es un servidor web de código libre disponible para la mayoría de las variantes de Unix. Es simple, rápido y seguro. Ideal 
para servir grandes volúmenes de información estática, ya que utiliza los requerimientos mínimos de un servidor HTTP.
El uso que se le da es para obtener velocidad en la transferecia de archivos y reducir los gastos innecesarios para funciones que 
no son requeridas en el servidor.
Sin embargo no posee las aplicaciones que se pueden obtener de un software tal como Apache

Cherokee es un servidor web multiplataforma que tiene por objetivo de ser rápido y completamente funcional. Es de software libre 
bajo la GNU. Soporta tecnologías como FastCGI, SCGI, PHP, SSI, SSL... Suele ser una alternativa al servidor Nginx 

Node.js es un entorno de tiempo de ejecución multiplataforma para la capa del servidor basándose en el lenguaje de programación 
ECMAScript que fue creado con el enfoque de ser útil en la creación de programas de red áltamente escalables, como por ejemplo 
servidores web. Es similar al JEE de Java. Además no se ejecuta en un navegador, sino en el propio servidor.
